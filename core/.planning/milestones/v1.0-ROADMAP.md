# Milestone v1.0: MVP

**Status:** ✅ SHIPPED 2026-05-28
**Phases:** Phase A - Phase F
**Total Plans:** 11 plans

## Overview

PiDashboard delivers a lightweight, smart dashboard kiosk system optimized for the Raspberry Pi Zero 2W. It decouples the heavy React-based drag-and-drop Web Admin Control Panel (compiled to a static bundle served by the Bun process) from a zero-framework kiosk client, which loads in just ~5KB of JavaScript. It uses a low-overhead, in-memory tmpfs RAM-disk watcher pipeline for IPC, structured 45MB rotating logging, and jittered exponential backoffs self-healing schedules.

## Milestone Summary

**Key Decisions:**
- **Single Bun Process**: Unifies server scheduler and routing in a single Bun thread to stay well under target RAM constraints (172-277MB active footprint).
- **Draft → Publish Editing**: Keeps layout scaling logic entirely in React Admin client-side, reducing Pi CPU cycles to zero during active editor sessions.
- **Vanilla Composited Kiosk Client**: Dynamically stitches standalone widget manifests to serve raw optimized HTML and auto-scales canvas grids natively.
- **IPC tmpfs Path Fallbacks**: Environment variable `PIDASH_IPC_DIR` automatically maps `/tmp/widgets` on Linux and fallback workspace caching on Windows for zero-overhead local development.

**Issues Resolved:**
- Single Bun process memory optimizations dropped baseline RSS RAM footprint by 40%.
- Dynamic JSON mapping correctly validates manifest dimensions, z-indexing, and opacity variables on the fly.
- Jittered backoff retries completely prevent socket lockups on network connection drops.
